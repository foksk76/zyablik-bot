# ADR-0001: Формат AI-assisted разработки

## Статус

Принято.

## Контекст

Проект небольшой, но его удобно развивать с участием AI-агентов: быстро вносить изменения в документацию, проверять webhook-скрипт, поддерживать однообразную структуру и не терять границы первого этапа.

При этом есть риск, что AI-агент начнет расширять проект за пределы текущей задачи: добавит SIEM, AI-аналитику, отдельный backend или хранение чувствительных параметров.

## Решение

Использовать в репозитории явный AI-каркас:

- `AGENTS.md` — главные инструкции для Codex и других агентов;
- `.agents/project-context.md` — краткий контекст проекта;
- `.agents/tasks/` — текущая задача и backlog;
- `.agents/checklists/` — обязательные проверки;
- `.agents/prompts/` — шаблоны постановки задач;
- `DEVELOPMENT.md` — общий процесс разработки;
- `scripts/verify-repo.sh` — базовая локальная проверка.

## Последствия

Плюсы:

- агент быстрее входит в контекст;
- меньше риск случайно добавить лишнюю функциональность;
- проще проверять изменения;
- документация становится частью процесса разработки.

Минусы:

- появляется несколько служебных файлов;
- их нужно поддерживать в актуальном состоянии.

## Правило

Если задача меняет архитектуру или границы проекта, сначала создается новый ADR, затем вносятся изменения.
