# Contributing

Проект небольшой, поэтому процесс максимально простой.

## Перед изменениями

- прочитать `README.md`;
- прочитать `AGENTS.md`;
- проверить текущую задачу в `tasks/sprints/`;
- убедиться, что изменение не выходит за границы текущего этапа.

## Требования к изменениям

- один PR или коммит — одна понятная задача;
- реальные секреты и внутренние значения не добавлять;
- организационные названия не использовать;
- изменения в поведении webhook сопровождать обновлением документации;
- крупные архитектурные решения фиксировать через ADR.

## Проверка

Перед завершением работы выполнить:

```bash
npm test
```

Если тесты недоступны, пройти чек-листы вручную:

- `.agents/checklists/pre-commit.md`;
- `.agents/checklists/security.md`.
