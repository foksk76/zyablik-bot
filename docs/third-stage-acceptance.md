# Критерии завершения третьего этапа

Документ фиксирует границы и критерии завершения третьего этапа проекта.

Третий этап начинается после принятого второго этапа и реализует MVP `MAX Identity Bot` на основе ADR-0005.

```text
ADR: docs/decisions/ADR-0005-use-hubot-for-max-identity-bot-mvp.md
Основной путь: Hubot-based MVP
Fallback: Node-RED workflow-прототип
```

## Цель третьего этапа

Реализовать минимальную модульную bot-platform для МАХ:

```text
входящее сообщение боту МАХ -> определение идентификатора получателя -> ответ с параметрами для Zabbix
```

Платформа должна быть готова к последующему расширению через плагины, но третий этап ограничен identity-сценарием.

## Граница третьего этапа

Входит:

- каркас модульной bot-platform;
- транспорт МАХ для приема входящих событий;
- транспорт МАХ для отправки ответов;
- identity plugin для определения типа получателя;
- обезличенные примеры входящих событий;
- локальный test harness;
- документация по запуску стенда;
- проверка стенда в WSL и/или LXC;
- описание применения Codex agent или аналога;
- unit-тесты и CI-прогон.

Не входит без отдельного ADR:

- изменение текущего Zabbix Media type `MAX`;
- изменение `src/zabbix-media-type/max-webhook.js`;
- автоматическая настройка Zabbix;
- хранение реальных идентификаторов получателей в репозитории;
- база данных;
- очередь сообщений;
- журнал доставки;
- автоматическая повторная отправка;
- маршрутизация уведомлений вне Zabbix;
- обработка инцидентов из мессенджера;
- AI-логика внутри runtime бота;
- промышленный эксплуатационный режим.

## Минимальная целевая структура

```text
bot-platform/
  core/
    config
    logger
    plugin-loader
    event-router
  transports/
    max/
      inbound-webhook
      outbound-client
      event-normalizer
  plugins/
    identity/
      handler
      formatter
  test-fixtures/
    max-inbound-user.json
    max-inbound-chat.json
```

## Acceptance criteria

- [ ] Создан каркас модульной bot-platform.
- [ ] Реализован MAX inbound transport для приема входящих событий.
- [ ] Реализован MAX outbound transport для отправки ответа.
- [ ] Реализован identity plugin.
- [ ] Для личного сообщения бот формирует параметры получателя для Zabbix.
- [ ] Для группового чата бот формирует параметры получателя для Zabbix.
- [ ] Секреты и реальные идентификаторы не попадают в репозиторий.
- [ ] Конфигурация вынесена во внешнее окружение или локальный файл, не предназначенный для коммита.
- [ ] Есть обезличенные fixtures для входящих событий МАХ.
- [ ] Есть локальный прогон обработки fixtures без обращения к реальному API.
- [ ] Есть интеграционный прогон с тестовым ботом или явно зафиксировано, почему он отложен.
- [ ] Проверена возможность запуска стенда в WSL или LXC.
- [ ] Описан режим работы Codex agent или аналога со стендом.
- [ ] `npm test` проходит локально или в CI.
- [ ] GitHub Actions green после изменений третьего этапа.
- [ ] Текущий Zabbix Webhook не изменен.

## Verification

- [ ] Проверить структуру каталогов bot-platform.
- [ ] Проверить обработку обезличенного user fixture.
- [ ] Проверить обработку обезличенного chat fixture.
- [ ] Проверить определение типа получателя.
- [ ] Проверить, что transport не логирует секреты.
- [ ] Проверить, что реальные идентификаторы не присутствуют в репозитории.
- [ ] Проверить, что `src/zabbix-media-type/max-webhook.js` не изменен.
- [ ] Проверить `npm test`.
- [ ] Проверить GitHub Actions.

## Ожидаемый итог

```text
Реализован MVP MAX Identity Bot на модульной bot-platform. Бот принимает входящие сообщения МАХ, определяет тип получателя и возвращает безопасную подсказку для настройки Zabbix. Стенд проверен в выбранной среде WSL или LXC. Текущий Zabbix Webhook остается без изменений.
```
