## Цель второго этапа

Выбрать технический подход для дальнейшего развития проекта в сторону модульной платформы взаимодействия с ботом МАХ.

Первый прикладной сценарий второго этапа:

```text
Пользователь или групповой чат отправляет сообщение боту МАХ.
Бот возвращает chat_id или user_id.
Полученное значение используется для настройки получателя уведомлений в Zabbix.
```

## Граница второго этапа

Второй этап является исследовательско-проектным.

Входит:

- исследование сценария получения `chat_id` / `user_id` через входящие сообщения бота МАХ;
- поиск open source кандидатов для модульной bot-platform или workflow-прототипа;
- сравнение вариантов: собственный сервис, готовый open source framework, workflow-platform;
- выбор рекомендуемого подхода для MVP `MAX Identity Bot`;
- подготовка ADR перед началом реализации нового сервиса, runtime или входящих webhooks;
- уточнение границ будущего MVP.

Не входит без отдельного ADR:

- реализация промышленного bot-service;
- запуск нового runtime;
- прием входящих webhooks от МАХ в продуктивном режиме;
- автоматическое изменение настроек Zabbix;
- интеграция с Zabbix API;
- очередь сообщений;
- база данных;
- журнал доставки;
- автоматическая повторная отправка;
- маршрутизация уведомлений вне Zabbix;
- обработка инцидентов из мессенджера;
- AI-логика.

## Критерии завершения второго этапа

Второй этап считается завершенным, когда выполнены условия:

- [x] Описан и согласован целевой сценарий MVP `MAX Identity Bot`.
- [x] Выполнен поиск в открытых источниках по open source bot/workflow framework кандидатам.

## Критерии качества выбора платформы

Выбранный подход оценен по критериям:

| Критерий | Итог |
|---|---|
| Security | токены и реальные идентификаторы не хранятся в репозитории; для MVP нужна внешняя конфигурация |
| Self-hosted | Hubot и Node-RED поддерживают локальное развертывание |
| Simplicity | Hubot выбран для identity-only MVP без базы данных, очереди и журнала доставки |
| Maintainability | Hubot дает понятную модель scripts/adapters; Node-RED оставлен как простой fallback-прототип |
| Extensibility | Hubot поддерживает расширение через scripts и adapter model |
| MAX compatibility | для Hubot нужен custom MAX adapter; для Node-RED fallback возможен HTTP/custom node |
| Zabbix compatibility | текущий Zabbix Webhook не меняется; Zabbix API не используется на MVP |
| License | Hubot MIT; Node-RED Apache-2.0; Errbot GPL-3 оставлен запасным кандидатом; n8n отклонен для open source shortlist |
| Operational risk | основной риск — новый runtime и custom MAX adapter; зафиксировано в ADR |

## Правило перехода к реализации

Реализация начинается только после завершения исследования и принятия ADR, если выбранный подход добавляет новый компонент.

До отдельной задачи реализации запрещено добавлять:

```text
новый сервис
новый runtime
обработчик входящих webhooks от МАХ
хранение состояния
журнал доставки
повторную отправку
маршрутизацию вне Zabbix
интеграцию с Zabbix API
```
