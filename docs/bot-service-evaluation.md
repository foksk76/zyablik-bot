# Оценка необходимости отдельного bot-service

Документ фиксирует результат оценки Task 9 после подтверждения базового сценария Zabbix -> МАХ.

## Вывод

На текущем этапе отдельный bot-service не требуется.

Прямой Zabbix Webhook достаточен для подтвержденного сценария доставки уведомлений Zabbix -> МАХ.

Реализация отдельного сервиса откладывается до появления требований к обработке входящих сообщений бота, в том числе для автоматизированного получения `chat_id` / `user_id` и дальнейшей настройки получателей уведомлений в Zabbix.

## Почему прямого webhook достаточно сейчас

- Доставка тестового сообщения подтверждена.
- Доставка Problem и Recovery подтверждена через Zabbix Action.
- Получение и настройка `chat_id` / `user_id` описаны как ручной административный процесс.
- Повторное создание или перенос Media type описаны в документации.
- Текущий сценарий не требует приема сообщений от пользователя в МАХ.
- Текущий сценарий не требует отдельной очереди, базы данных или журнала доставки.

## Когда вернуться к bot-service

Отдельный сервис стоит рассмотреть, если появятся требования:

- принимать входящие сообщения от пользователей или групповых чатов;
- автоматически возвращать `chat_id` / `user_id` для дальнейшей настройки получателей в Zabbix;
- хранить журнал попыток доставки;
- выполнять повторную отправку при временных ошибках API;
- централизованно управлять маршрутами уведомлений вне Zabbix;
- выполнять дополнительную диагностику доставки вне стандартных логов Zabbix.

## Сравнение вариантов

| Критерий | Прямой Zabbix Webhook | Отдельный bot-service |
|---|---|---|
| Простота внедрения | Высокая | Ниже |
| Количество компонентов | Минимальное | Больше |
| Поддержка текущего сценария доставки | Достаточная | Избыточная |
| Получение `chat_id` / `user_id` | Ручной административный процесс | Можно автоматизировать через входящие сообщения бота |
| Журнал доставки | Стандартные средства Zabbix | Можно реализовать отдельно |
| Повторная отправка | Ограничена возможностями текущего подхода | Можно реализовать явно |
| Нужен ADR перед реализацией | Нет | Да |

## Решение на текущий этап

Код webhook не меняется.

Новые компоненты не добавляются.

Отдельный bot-service не реализуется до появления подтвержденных требований к обработке входящих сообщений бота или другим возможностям, которые нельзя закрыть прямым Zabbix Webhook без неоправданного усложнения.

Если такие требования появятся, перед реализацией создается ADR в `docs/decisions/`.
