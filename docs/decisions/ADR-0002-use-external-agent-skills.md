# ADR-0002: Использовать внешний набор agent-skills без submodule

## Статус

Принято.

## Дата

2026-07-07

## Контекст

Проект ведется с участием AI-агентов и Codex. Для повышения качества разработки полезно использовать готовые инженерные workflows: постановка задачи, планирование, инкрементальная реализация, тестирование, review, безопасность и ведение документации.

Внешний репозиторий `https://github.com/addyosmani/agent-skills` содержит такие workflows в формате Markdown skills.

Нужно подключить этот набор так, чтобы:

- не усложнять маленький репозиторий;
- не тащить внешний код и историю в проект;
- сохранить возможность работы в OpenAI chat, Codex CLI, Codex IDE и других средах;
- не нарушать правило отсутствия чувствительных данных и лишних зависимостей.

## Решение

Не добавлять `agent-skills` как git submodule.

Использовать внешний репозиторий как источник правил и workflow:

```text
https://github.com/addyosmani/agent-skills
```

В проекте хранить:

- ссылку на внешний репозиторий;
- список рекомендуемых skills;
- правила применения в `AGENTS.md`;
- инструкцию в `docs/agent-skills-integration.md`;
- игнорируемые пути для локальной копии skills.

Для документации и ADR использовать подход `documentation-and-adrs`.

## Рассмотренные альтернативы

### Git submodule

Плюсы:

- внешний набор всегда рядом с проектом;
- можно работать офлайн после клонирования с submodules.

Минусы:

- усложняет простой проект;
- требует отдельного обновления submodule;
- добавляет лишний шум для текущего этапа.

Решение: отклонено для текущего этапа.

### Локальная копия в игнорируемом каталоге

Плюсы:

- удобно для Codex CLI и IDE;
- не попадает в историю проекта;
- можно обновлять локально по необходимости.

Минусы:

- каждый разработчик настраивает окружение сам.

Решение: принято как допустимый локальный вариант.

### Ссылка и выборочное использование skills

Плюсы:

- минимально для репозитория;
- хорошо подходит для OpenAI chat;
- можно загружать только нужный `SKILL.md`.

Минусы:

- требует дисциплины агента и явных правил.

Решение: принято как основной вариант.

## Последствия

- `AGENTS.md` становится основным местом правил для AI-агентов.
- Документация по подключению skills хранится в `docs/agent-skills-integration.md`.
- Локальные копии skills не коммитятся.
- Архитектурные решения и существенные изменения документации ведутся через ADR в `docs/decisions/`.
- `documentation-and-adrs` применяется как основной подход к README, ADR, changelog и фиксации контекста для будущих инженеров и агентов.
