# Ручной прогон Zabbix Media type MAX

Документ описывает ручную проверку Task 2: отправку тестового сообщения из Zabbix Media type `MAX` на тестового получателя МАХ.

Это не автоматический тест и не решение о test harness. Автоматизацию тестирования нужно оформлять отдельным ADR.

## Цель

Проверить, что текущая настройка Zabbix Media type `MAX` может выполнить отправку тестового уведомления на тестового получателя.

## Что проверяем

- Media type `MAX` создан как `Webhook`.
- В Media type используется скрипт из `src/zabbix-media-type/max-webhook.js`.
- Параметры заполнены по `docs/zabbix-media-type.md`.
- Тестовое сообщение отправляется через встроенный тест Media type в Zabbix.
- Результат фиксируется без чувствительных значений.

## Что не проверяем

- Problem/Recovery сценарии.
- Автоматическую генерацию событий Zabbix.
- Повторную отправку.
- Журнал доставки.
- Отдельный bot-service.
- Zabbix container.
- Локальный test harness.

## Перед началом

Проверить, что подготовлены:

```text
- тестовый получатель МАХ;
- тестовое значение получателя для параметра To;
- рабочее авторизационное значение бота МАХ;
- доступ к Zabbix с правом тестирования Media type;
- исходящий доступ Zabbix к MAX Bot API;
- HTTPProxy, если исходящий доступ выполняется через прокси.
```

Чувствительные значения не записывать в репозиторий.

## Параметры теста

Использовать структуру параметров из:

```text
docs/zabbix-media-type.md
examples/media-params.md
```

Обезличенный пример:

```text
APIUrl: https://platform-api2.max.ru/messages
HTTPProxy:
Message: Тестовое уведомление из Zabbix
ParseMode: HTML
RecipientType: chat_id
Severity: High
Subject: TEST Zabbix -> MAX
To: test-recipient-id
Token: задается только в Zabbix
Trigger_status: PROBLEM
```

## Порядок выполнения

1. Открыть в Zabbix Media type `MAX`.
2. Проверить, что тип Media type — `Webhook`.
3. Проверить, что в поле скрипта используется актуальный код из `src/zabbix-media-type/max-webhook.js`.
4. Открыть тест Media type.
5. Заполнить тестовые значения параметров.
6. Выполнить тест отправки.
7. Проверить результат в тестовом получателе МАХ.
8. При ошибке открыть лог теста Media type в Zabbix.
9. Зафиксировать обезличенный результат в этом документе или в отдельной заметке этого раздела.

## Фиксация результата

Заполнять только обезличенные данные.

```text
Дата прогона:
Исполнитель:
Zabbix version:
Media type: MAX
Recipient type: chat_id / user_id
HTTPProxy used: yes / no
Result: success / failed
HTTP status: <без чувствительных значений>
Error summary: <кратко, если есть>
Message delivered: yes / no
Notes:
```

## Критерии закрытия Task 2

Task 2 можно закрыть только после фактического ручного прогона.

Минимальные условия:

```text
- тест Media type выполнен в Zabbix;
- результат доставки проверен в тестовом получателе МАХ;
- при ошибке зафиксированы HTTP-код, краткий текст ошибки и место диагностики;
- в репозиторий не внесены чувствительные значения;
- документация обновлена только по фактически проверенному поведению.
```
