# Пре-продакшн: бренд «Зяблик» + лицензия Apache-2.0 + полный ренейминг

> Идея. Не ADR, не спецификация, не задача. Отправная точка для будущих ADR и
> декомпозиции в `tasks/sprints/`. Создана через навык `idea-refine`.

## Problem Statement

Как нам провести пре-продакшн эксплуатацию бота в корпоративной сети с
брендингом «Зяблик/Zyablik», выбрать и оформить open source лицензию, и
переименовать проект из `zabbix-max-alert-bot` в `zyablik-bot` — всё
одновременно, одним этапом, до запуска на весь отдел?

Текущее состояние: проект не имеет лицензии (all rights reserved),
техническое имя `zabbix-max-alert-bot` не подходит для пользовательского
бренда, нет визуальной идентичности (логотип готов, но не загружен).
Проект планируется как публичный open source, репо сейчас приватный на
GitHub.

## Recommended Direction

**Полный ренейминг + лицензия Apache-2.0 + бренд «Зяблик» — один ADR,
один PR, один этап.**

### Почему Apache-2.0

| Юрисдикция | Обоснование |
|---|---|
| **РФ** | Признаётся судами. Основа для включения в реестр Минцифры. Ст. 1271 ГК РФ — указание автора в LICENSE |
| **EU (CRA)** | SPDX-рекомендуемая. Art. 12(1) CRA освобождает open source от требований безопасности |
| **US** | Признана судами (Jacobsen v. Katzer, 2008). Art. 3 — patent license от автора |

**Почему не MIT:** Нет патентной защиты. Для публичного open source от
физического лица — insurance от patent trolls.

**Почему не GPL-3.0:** Copyleft отпугивает корпоративных пользователей,
которые хотят форкнуть и адаптировать под себя. Для инфраструктурного
инструмента это важнее, чем для прикладного софта.

**Совместимость с зависимостями:** jose (MIT), @okta/jwt-verifier (ISC),
better-sqlite3 (MIT) — все совместимы с Apache-2.0.

### Почему полный ренейминг сейчас

Проект уходит в публичный open source. Техническое имя `zabbix-max-alert-bot`
описывает только Zabbix-кейс. Название `zyablik-bot` — нейтральное, подходит
для multi-source платформы (ADR-0022). Ренейминг до публикации проще, чем
после — нет внешних ссылок, которые нужно перенаправлять.

## Что входит в этап

### 1. Лицензия

Два файла лицензии: английский (`LICENSE`) и русский (`LICENSE.ru`). Русская
версия — официальный перевод Apache-2.0 Foundation, необходим для
юридической определённости в РФ (ст. 1271 ГК РФ: «Автором признается
лицо, указанное в экземпляре программы»).

| Файл | Действие |
|---|---|
| `LICENSE` (новый) | Текст Apache-2.0, copyright `Copyright 2026 Kirill Fomichev` |
| `LICENSE.ru` (новый) | Официальный перевод Apache-2.0 на русский (Apache-2.0 Foundation) |
| `package.json` | Поле `"license": "Apache-2.0"` |
| `README.md` | License badge + секция «License» |
| Все `.js` файлы (~25) | SPDX-header: `// SPDX-License-Identifier: Apache-2.0` |
| `CONTRIBUTING.md` | Строка: `By contributing, you agree to license your contributions under Apache-2.0` |

## MVP Scope

Этап завершён:
- Проект имеет лицензию Apache-2.0 во всех файлах
- Проект переименован в `zyablik-bot` на всех уровнях
- README.md — «посадочная страница» для получателей с логотипом
- `npm test` проходит без ошибок
- Все доки согласованы друг с другом

**Не является MVP:** деплой на сервер, настройка Zabbix Media type под
новым именем, миграция systemd-сервисов на стенде.

## Not Doing (and Why)

- **Миграция systemd на стенде сейчас** — стенды dev/stand/staging могут
  использовать старые имена до следующего релиза. Ренейминг сервисов в
  репо — это источник новых имен, не миграция.
- **Включение в реестр Минцифры** — это отдельный процесс, требующий
  юридического сопровождения. Лицензия Apache-2.0 — необходимое, но не
  достаточное условие.
- **Юридическое оформление CLA** — для проекта от физического лица в
  публичном open source CLA избыточен. `CONTRIBUTING.md` с grant-строкой
  достаточен.
- **Смена Git remote на внутренний хостинг** — сейчас GitHub, внутренний
  хостинг потом. Ренейминг не зависит от места хранения.
- **Публикация в npm** — `"private": true` остаётся. Публикация — отдельная
  история.
- **Обновление CHANGELOG.md** — changelog отслеживает релизы, не рефакторинг.
  Ренейминг попадёт в CHANGELOG при следующем релизе.
